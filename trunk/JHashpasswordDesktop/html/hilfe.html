Wie es zu dem Programm kam

Die Idee zu diesem Programm kam meinem Vater durch einen Bericht in den Heise News von 2004-08-09 (http://www.heise.de/newsticker/meldung/49900).

Die meisten Menschen können sich keine große Zahl unterschiedlicher Passwörter für unterschiedliche Zwecke merken und benutzen dann das selbe Passwort mehrmals. Das macht das Passwort anfällig für das sog. Phishing. Die Bekanntgabe eines Passworts an einer unverdächtigen Stelle erlaubt dann die Verwendung an einer Stelle, wo Sie das gar nicht mögen (z. B. Homebanking).

Eine gute Idee wäre ein einziges gutes Passwort (besser eine Passphrase; im Folgenden bezeichnet Passphrase das, was Sie sich merken müssen, Passwort dagegen das, was der Host zu sehen bekommt) irgendwie einzigartig für verschiedene Hosts zu machen und zwar so, dass die Kenntnis des einen Passworts selbst bei Kenntnis aller anderen Informationen (außer der Passphrase selbstverständlich) keinen Schluss auf ein anderes Passwort erlaubt. Trotzdem soll man sich aber nur eine (gute) Passphrase merken müssen.


Wie das?

Hashing!

Hashing erzeugt einen Einbahncode eines Strings. Einbahncode heißt, dass aus dem Ergebnis (dem Code) der Eingabe-String nicht zurück erschlossen werden kann, aber derselbe String immer den selben Code erzeugt.

Bei guten, kryptographischen Hashalgorithmen erzeugt selbst die kleinste Änderung im String ein völlig unterschiedliches Ergebnis. Wenn wir also die Passphrase um einen (abgespeicherten) Bezeichner für den Host und auch noch um einen (abgespeicherten) Bezeichner für den Login-Namen ergänzen, erhalten wir für jeden Host und jeden Login-Namen einen völlig neuen Code. Und selbst aus der Kenntnis mehrerer Codes und der abgespeicherten Daten, lässt sich die Passphrase nicht errechnen.

Den Code übersetzen wir nun wieder in einen String, das Passwort, das ein sehr gutes Zufallsmuster hat, da er dem Bitmuster des nicht erratbaren Ergebnisses entspricht (das erreichen wir indem wir quasi eine Zahl zur Basis N erzeugen, wobei wir einen bestimmten (abgespeicherten) Characterset mit N Zeichen als erlaubte "Ziffern" benutzen).


Hashing - Voilá!

Das ist es, was dieses Programm tut: Es nimmt Ihre Passphrase, die Sie sich leicht merken können (es sollte trotzdem eine gute Passphrase sein; s. z. B. 'diceware') fügt einen von Ihnen angegebenen String für den Host und für den Login-Namen hinzu und erzeugt einen Hashcode (wahlweise MD5, SHA1, SHA256, SHA384, oder SHA512) des Ergebnisstrings. Diesen Code verwandelt es dann mit dem von Ihnen bestimmten Zeichensatz in einen neuen Quasi-Zufallsstring um, der das eigentliche Passwort darstellt und kopiert das Ergebnis wenn möglich in der vollen notwendigen Länge resp. auf die maximal erlaubte Länge gekürzt in die Zwischenablage.

Die von Ihnen angegebene Länge des eigentlichen Passworts ist eine Obergrenze, die von dem tatsächlich erzeugten Passwort auch unterschritten werden kann! Dennoch ist dieses Passwort bei voller Länge eine Quasizufallszahl mit 128 (MD5) oder 160 (SHA1) oder der angegebenen Zahl Bits (SHA###) und ebenso sicher. Sollte aber die von Ihnen vorgegebene Länge die eigentlich notwendige Stellenzahl seinerseits unterschreiten, ist die Sicherheit natürlich auf <zeichenzahl_buchstabensatz>  <stellenzahl> begrenzt (z.B. 64 Zeichen im Buchstabensatz, 10 Zeichen Passwortlänge = 1.152.921.504.606.846.976 mögliche Kombinationen - auch nicht schlecht). Sie können übrigens also auch bei eingeschränkter Länge des zulässigen Passworts die Sicherheit durch Hinzufügen von möglichst vielen Zeichen zum Zeichensatz erhöhen (so der Host die Zeichen zulässt...). Falls Ihr Host 32 Zeichen Länge zulässt können Sie ruhig den vorgegebenen Zeichensatz bis SHA256 beibehalten. Er reicht dann immer aus, das Bitmuster des Codes vollständig zu repräsentieren.

Wenn Sie möchten, finden Sie den Knopf "Buschstabensatz zufällig mischen". Wenn Sie diesen benutzen, wir Ihr Zeichenvorrat in eine zufällige Reihenfolge gebracht. Seien Sie vorsichtig und benutzen Sie diesen Knopf nur einmal, wenn Sie einen neuen Login anlegen (oder wenn sie so ein neues Passwort erzeugen wollen)! Wenn Sie es später tun, können Sie mglw. Ihr Passwort nicht wieder herstellen! Dasselbe gilt natürlich auch, wenn Sie den Zeichenvorrat manuell verändern!



Wie - wo - was?

Alle Provider/Hosts Namen werden genau so, wie Sie sie eingegeben haben, wie gesagt in einer XML-Datei abgelegt. Dazu wird die von Ihnen angegebene Passwortlänge, der Buchstabensatz und der Hashtyp gespeichert. Wenn Sie mehrere Logins beim gleichen Host haben, werden die auch gemerkt. Das eigentliche Passwort wird aber jeweils erst zur Laufzeit mit Hilfe der von Ihnen einzugeben Passphrase neu erzeugt und in die Zwischenablage kopiert. Das eigentlichen Passwort wird also normalerweise nie sichtbar und ist somit auch nur schwer auszuspähen.

Ein echter Sicherheitsaspekt ist dabei, dass egal was Sie eingeben eine Passwort erzeugt wird. Sollte ein Angreifer Ihre XML-Datei und mein (dieses) Programm haben (wobei letzteres einfach ist), muss er jedes erzeugte Passwort direkt beim Provider (Host) ausprobieren. Typischerweise wird der Provider (Host) das bemerken und bei mehrfacher Falscheingabe mit einer Sperrfrist reagieren. Dann kann ein Brute-Force-Angriff ziemlich langwierig werden.

Dieses Programm speichert zu jedem Login individuell den verwendeten Hash-Algorithmus, die Passwortlänge und den erlaubten Zeichensatz in die XML-Datei. Dadurch ist das Programm noch flexibler.

Die aktuelle Version fügt dem Ganzen noch die Hash-Methoden SHA256, SHA384 und SHA512 und die Möglichkeit die Reihenfolge des Zeichensatzes zu verwürfeln hinzu. (Natürlich dürfen Sie das nicht jedes Mal sondern nur beim ersten Mal wenn Sie einen neuen Login anlegen tun, sonst werden sie nicht mehr dasselbe Passwort erhalten. Da die meisten wohl einfach den vordefinierten 64-Zeichen Vorrat übernehmen, wollte ich so ein wenig mehr Unterschied hinein bringen, da so also praktisch jeder Login seinen eigenen Zeichensatz benutzen kann. Es gibt immerhin 1,268869x1089  Permutationen, was das Angreifen ohne XML-File weiter deutlich erschwert.)

Einige Programme erlauben nicht die Übernahme eines Passworts aus der Zwischenablage (soweit mir bekannt, betrifft das nicht die Web-Browser, aber mglw. möchten Sie mein Programm auch anderweitig einsetzen...). Dazu haben sie einen Button, um dass erzeugte Passwort anzuzeigen und müssen es abtippen.

Theoretisch können Sie mein Programm und die XML-Datei auf einem Memorystick mitnehmen und auch in einem Internet-Cafe verwenden.
